화면 정보

  • ADM-CHAT-001 · 챗봇 대화 내역

  •    • 화면 유형: 어드민 목록(조회 전용) + 대화 로그 팝업(오버레이)
       • 템플릿: 풀무원 디자인밀 (PC 고정)
       • 경로: Admin > 챗봇 관리 > 챗봇 대화 내역
       • 소스: ADM-CHAT PRD (세부기능정의 워크북 기반)
       * GNB 메뉴 위치(챗봇 관리 그룹)는 워크북 미정의 — (제안)

UI 요소

  • 1. 상단 유틸리티 바
       • 좌: DESIGN MEAL 로고 + Admin Mode
       • 우: 관리자(admin) · Dark Mode · View Site · Log Out

  • 2. 좌측 GNB
       • 활성: 챗봇 관리 → 하위 <챗봇 대화 내역> 선택
       • 메뉴 접근 권한 보유 어드민 계정만 진입 가능

  • 3. 페이지 헤더
       • 타이틀 "챗봇 대화 내역"
       • 진입 시 필터 기본값으로 자동 조회 (기간=당일 · 그 외 전체 · 최신순)

  • 4. 필터 조건값 (8종)
       • 4-1. 날짜(기간): 데이트피커, YYYY-MM-DD ~ YYYY-MM-DD
          1) 클릭 시 기간선택 달력 출력 / Default: 당일
       • 4-2. 검색 기준 + 검색어
          1) 기준 셀렉박스 — Default: 검색어 전체 / 검색어 전체·고객 ID·대화 고유 ID·대화 식별 ID·사용 인텐트
          2) 검색어 인풋 — 플레이스홀더 "검색어를 입력해 주세요"
       • 4-3. 상담실 이관여부 — Default: 전체 / 전체·상담실 이관 Y·상담실 이관 N
       • 4-4. 미응답 여부 — Default: 전체 / 전체·미응답 Y·미응답 N
       • 4-5. 구매 전환 여부 — Default: 전체 / 전체·구매 전환 Y·구매 전환 N
       • 4-6. 대화 상태 — Default: 전체 / 전체·대화 진행중·사용자 완료·시스템 완료
       • 4-7. 확인 결과 — Default: 전체 / 전체·미확인·정답·오답·모호
       • 4-8. 답변 피드백 — Default: 전체 / 전체·긍정·부정·미답변

  • 5. [검색] / [초기화] 버튼
       • [검색] 클릭 또는 ENTER 시 필터 조건값으로 조회 실행
       • [초기화] 클릭 시 선택/입력된 필터 조건값 초기화
       * 초기화 후 재조회 여부는 워크북 미정의 — 조건값 리셋만 수행

  • 6. 리스트 솔팅
       • Default: 최신순 / 최신순·오래된순
       • 변경 시 테이블 재정렬

  • 7. 엑셀 다운로드 버튼
       • 클릭 시 현재 조회된 테이블 엑셀 다운로드 실행

  • 8. 대화 내역 테이블 (15컬럼)
       • 8-1. No: 1부터 오름차순
       • 8-2. 대화 고유 ID (예: 5b000af6-fbd4-40b1-8323-d34cea0e76fa)
       • 8-3. 대화 식별 ID (예: 260304-095502-lyg701)
       • 8-4/8-5. 시작·종료일시: YY-MM-DD HH:MM:SS, 세션 종료 전 "-"
       • 8-6. 고객 ID / 8-7. 대화 상태(진행중·사용자 완료·시스템 완료)
       • 8-8. 총 시간: HH:MM:SS / 8-9. 메시지 수: 발화 없으면 0
       • 8-10. 사용 인텐트: 해당 없으면 "-" (예: NDP문의)
       • 8-11. 상담실 이관 여부: Y/N
       • 8-12. 미응답 여부: 미응답 시 Y(건수), 예: Y(1) / 없으면 N
       • 8-13. 구매 전환 여부: 추천 상품 → 장바구니 이동 시 Y, 그 외 N
       • 8-14. 확인 결과: 미확인·정답·오답·모호
       • 8-15. 답변 피드백: 긍정·부정·-
       • 각 행 클릭 시 대화 로그 팝업 출력 (⑩~⑭)

  • 9. 페이지네이션
       * 워크북 미정의 — (제안) 10건/페이지, 이전·다음·번호 버튼

DIALOG — 대화 로그 팝업

  • 10. 대화 상세 정보
       • 고객 ID / 대화 고유 ID / 대화 식별 ID / 메시지 수 / 상담실 이관 여부 / 미응답 여부 / 사용 인텐트 / 대화 상태: 테이블 데이터와 동일하게 출력
       • 총 시간: YY-MM-DD HH:MM:SS ~ YY-MM-DD HH:MM:SS (총 시간 HH:MM:SS)

  • 11. 클립보드 복사
       • 대화 고유 ID / 대화 식별 ID 우측에 클립보드 아이콘 노출
       • 아이콘 클릭 시 ID 복사 처리 + 토스트 "텍스트가 복사되었습니다." 노출

  • 12. API호출이력
       • 대화 중 호출된 API 출력 (예: 주문내역 API)

  • 13. 턴별 대화 로그
       • 턴 구성: 사용 인텐트 / 발화 시점(YY-MM-DD HH:MM:SS) / 사용자 발화 메시지 / 챗봇 처리 모듈 / 챗봇 답변 메시지
          1) 처리 모듈: 시나리오 / LLM / RETRIEVAL / 상담실 / 가드레일
          2) 대화 중 API 호출된 경우 호출된 API 출력

  • 14. 팝업 닫기
       * 워크북 미정의 — (제안) X 버튼 + 배경 클릭 + ESC 키 / 닫기 후 목록 유지

동작 / 예외

  • 진입 시 기본 조회
       • 필터 기본값(기간=당일 · 전체 · 최신순)으로 대화 내역 조회 API 호출

  • 조회 결과 0건
       * (제안) 빈 목록 상태 — "조회된 대화 내역이 없습니다."

  • 조회 API 실패
       * (제안) 에러 상태 + 재시도 유도 — "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해주세요."

  • 엑셀 다운로드 실패
       * (제안) 실패 토스트 — "다운로드에 실패했습니다. 다시 시도해주세요." / 목록 유지

  • 접근 권한 없음
       * (제안) 메뉴 접근 차단 — "접근 권한이 없습니다. 관리자에게 문의해주세요."

  • 데이터 표기 예외
       • 종료일시 없음(세션 종료 전) → "-"
       • 사용자 발화 없음 → 메시지 수 0
       • 해당 인텐트 없음 → "-"
       • 미응답 없음 → N / 발생 시 Y(건수)
       • 구매 전환 미발생 → N
